becto-fli4l: git-skeleton for fli4l-4

The last testing version of fli4l was released in 2022/10/14 → the read-only svn repository is currently still online → so here is a part of the original svn repository as a skeleton on github.

The skeleton helps you to create a fli4l installation folder from the latest svn revision (r60808) and is also a template for further development of the software on your own computer.

Since fli4l is designed as a router or firewall, essential components should only be downloaded from secure sources. For this reason, the skeleton only consists of the absolute minimum.

If the read-only svn server of fli4l fails, the software can still be further developed on your own (intel-based) computer at your own risk.

How to use the skeleton:

  • Clone the github repository becto-fli4l to get the skeleton

  • Use a svn checkout to load the precompiled binaries from the fli4l svn-repo https://repo.nettworks.org/svn/fli4l → a shell script is included in the skeleton
    → ./_svn-repo.nettworks.org~bin.sh ↵

  • Optional:

    • Use a svn checkout to load the original packages directory from the fli4l svn-repo https://repo.nettworks.org/svn/fli4l → a shell script is included in the skeleton
      → ./_svn-repo.nettworks.org~packages.sh ↵

    • Then you can move the packages you need into the package directory under /src/packages

  • Execute the shell script → ./_mkfli4lgit.sh ↵ in the src folder → done

Please note that the skeleton is still new and various fli4l packages and functions are still missing, e.g. precompiled latex docs.
